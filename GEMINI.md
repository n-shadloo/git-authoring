# Commit conventions

This repository's commit-message rules live in `AGENTS.md` at the repository root.
Follow it in full when writing any git commit here: pick an accurate Conventional
Commit type and scope, write an imperative subject, add a body only when it
explains *why*, notate breaking changes, and commit only when explicitly asked —
never via `git add -A`. See `AGENTS.md` for the complete workflow and `references/`
for deeper guidance.
